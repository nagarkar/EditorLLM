## Technical Audit instruction quality (score 0–5)

Evaluate whether the markdown is a **usable audit playbook** for the manuscript's **axioms, notation, and internal consistency**.

### Dimensions to check
- **Core axioms / framework** — explicit `##` coverage; bullets name *what* to verify, not hand-waving.
- **Notation & LaTeX** — captions, equation references, symbols: concrete pass/fail style checks.
- **Terminology & constants** — glossary-style rules where the manuscript demands precision.
- **Checklist usability** — another editor could run W2 from these bullets alone.

### Scoring guide

| Score | Criteria |
|:-----:|----------|
| **5** | All dimensions strong; bullets are specific and manuscript-aware. |
| **4** | One dimension slightly thin; still production-usable. |
| **3** | Covers axioms OR notation well; the other is shallow but present. |
| **2** | Mostly generic "check facts" prose; weak `##` structure. |
| **1** | Outline only; not operational. |
| **0** | Empty, incoherent, or evaluation error. |
